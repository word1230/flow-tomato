Project Brief: Flow Tomato 番茄钟
1. 核心愿景与氛围 (Core Vision & Vibe)
项目名称: Flow Tomato (心流番茄)
核心愿景: 打造一款极简、快速的Web端番茄钟应用。它不仅仅是一个计时器，更是一个帮助用户将复杂任务拆解、进入心流状态、战胜拖延的效率伙伴。
产品关键词: 专注 (Focus), 掌控感 (Sense of Control), 成就感 (Sense of Achievement)
产品氛围: 整体感觉应是沉静、清晰、令人安心的。它是一个不打扰用户的助手，通过流畅的设计和积极的反馈，帮助用户消除焦虑，激发“即刻行动”的动力。
2. 用户画像与场景 (User Persona & Scenario)
核心用户画像:
身份: 高强度脑力工作者，典型代表是需要处理多任务的程序员或需要进行深度学习的学生。
特点: 他们面对的是复杂、非线性的任务，经常被突发事件打断，渴望一个能帮助他们理清思路、重获秩序感的工具。
核心使用场景:
一位程序员面对着一个令人头疼的任务清单：紧急Bug修复、新需求文档撰写、准备会议、处理线上告警等。他感觉千头万绪，无从下手，导致了拖延。他打开 Flow Tomato，希望能将最棘手的“新需求开发”任务进行拆解和规划，并立即开始执行第一个步骤。
3. 核心功能与用户流程 (Core Functionality & User Flow)
功能清单 (Features)
无登录任务系统: 用户通过输入一个自定义的Key（任意字符串）来进入自己的空间，空间中包含任务列表等，数据存储于本地，免去注册登录流程。
主任务管理: 支持通过输入框回车快速添加主任务；支持标记完成、更新和删除任务。
思路规划区 (Idea Zone): 每个主任务都关联一个大文本编辑区。用户可以自由地在其中记录任务拆解的思路、步骤、笔记等。
番茄数预估: 用户可以直接为每个主任务输入一个数字，来设定完成它预估需要的番茄钟数量。
番茄钟计时器:
工作流程: 遵循标准的番茄工作法循环：25分钟工作 / 5分钟短休。每完成4个番茄钟，进入一个15分钟的长休息。
任务关联: 必须选择一个任务后才能启动番茄钟。
智能提醒系统:
在每个工作或休息时间段结束时，通过浏览器桌面通知和预设声音进行提醒。
提供设置选项，允许用户选择“工作结束后是否自动开始休息”。
灵活的任务完成机制:
当一个任务的预设番茄数全部完成后，系统会弹窗询问用户是否要将该任务标记为“已完成”。
用户可以随时手动将任务标记为“已完成”，当前番茄钟会立即停止。
番茄统计: 在界面的显眼位置（如顶部）实时显示用户累计完成的番茄总数。
核心用户流程 (User Flow)
进入: 用户打开应用，输入自己的Key，加载出他专属的任务列表。
规划 (Plan):
在任务输入框内输入“开发新用户登录功能”，按回车，创建成功。
点击该任务，右侧（或下方）的“思路区”被激活，用户开始输入：“1. 设计数据库表结构 2. 编写后端API接口 3. 实现前端登录页面...”
在任务旁边的数字输入区，输入8，代表预估需要8个番茄钟。
执行 (Execute):
点击任务旁的“开始”按钮，25分钟倒计时启动。
关键: 在计时期间，旁边的“思路区”内容保持可见，用户可以随时查看或修改。
时间到，桌面通知和声音响起。根据设置，自动或手动进入5分钟休息。
完成 (Complete):
当第8个番茄钟也完成后，系统弹窗提示：“已完成所有计划，是否标记任务为完成？”。
或者，用户在第6个番茄钟就提前完成了所有工作，他可以直接点击任务旁的“完成”勾选框，任务被划掉，计时结束。
4. 技术栈与关键约束 (Tech Stack & Key Constraints)
前端框架: 推荐 Svelte。因其高性能、轻量级和优秀的开发体验，与本项目的“快速、简约”气质完美契合。
数据存储: 使用浏览器 localStorage。所有数据以用户输入的Key为索引进行本地存储。
关键约束:
纯静态部署: 项目必须是纯前端应用，能够直接部署在 GitHub Pages 或 Cloudflare Pages 等静态托管平台。
零广告: 严格禁止任何形式的广告或用户追踪。
移动端适配: 必须进行响应式设计，确保在手机浏览器上也有优秀的使用体验。
5. 风格与交互参考 (Style & Interaction References)
视觉风格:
参考: Google AI Studio / Google Cloud Console。
色彩: 以无彩色系（白、灰、深灰）为基底，搭配一个单一、不刺眼的蓝色作为高亮和交互色。
布局: 极简主义，大量留白，确保信息层次清晰，无压迫感。
字体: 现代、清晰的无衬线字体 (e.g., Roboto, Inter)。
交互与动画:
核心原则: 快速、精准、不打扰。
主要交互: 所有状态变更（如点击按钮）都应即时响应，无延迟感。
点缀动画:
任务完成: 应用一个快速的划线动画和淡出效果。
计时器: 数字直接跳变，避免分散注意力的翻页等动画。
文本输入: “思路区”的文本光标（Caret）采用类似 VS Code 的平滑移动动画，提升长时间输入的舒适感。
6. 成功标准 (Success Criteria)
最终交付物:
一份完整的、结构清晰的前端项目源代码包（HTML/CSS/JS）。
代码应能直接通过git push推送到GitHub仓库，并由Cloudflare Pages等平台自动完成构建和部署。
验收清单 (Acceptance Checklist):

数据持久化: 用户可以通过输入自定义Key来加载/保存其任务数据到localStorage。

任务管理: 可以完整地执行添加、标记完成、删除主任务的操作。

思路区功能: 每个任务的“思路区”都能正常输入、编辑和保存。

思路区可见性: 在番茄钟倒计时界面，当前任务的“思路区”必须保持可见且可编辑。

番茄数设定: 可以为每个任务设定并保存预估番茄数。

计时器核心功能: 点击“开始”能启动25分钟倒计时，并与任务关联。

番茄钟循环: 计时器能正确地按“25/5”和4次循环后“15”分钟长休的模式工作。

提醒系统: 在工作和休息结束时，能成功触发桌面通知和声音提醒。

任务完成流程: 预设番茄数用完后，弹窗提示；用户也可随时手动完成任务。

数据统计: 页面能正确显示累计完成的番茄总数。

响应式设计: 应用在桌面和主流移动端浏览器上均无布局错乱，体验良好。

视觉风格: 应用的整体UI/UX符合“风格与交互参考”部分定义的标准。
